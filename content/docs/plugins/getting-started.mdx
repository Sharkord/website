# Getting Started

This guide walks through creating a minimal Sharkord plugin.

## 1) Create a Plugin Directory

Create a project folder, for example `my-plugin`.

## 2) Initialize the Package

```bash
bun init
```

## 3) Configure `package.json`

Your package must include standard fields and Sharkord plugin metadata.

Required:

- `name`
- `version` (semver)
- `sharkord.entry` (must point to a `.js` file)
- `sharkord.author`
- `sharkord.description`

Optional:

- `sharkord.homepage`
- `sharkord.logo`

Example:

```json
{
  "name": "my-plugin",
  "version": "0.0.1",
  "module": "src/index.ts",
  "sharkord": {
    "entry": "index.js",
    "author": "Me",
    "homepage": "https://some-page.com",
    "description": "This is my first Sharkord plugin!",
    "logo": "https://some-page.com/logo.png"
  },
  "type": "module",
  "scripts": {
    "build": "bun build src/index.ts --outdir dist --target bun --minify --format esm && cp package.json dist/"
  },
  "devDependencies": {
    "@types/bun": "latest"
  },
  "peerDependencies": {
    "typescript": "^5"
  }
}
```

## 4) Install the SDK

The SDK is not currently published to a package registry. Use `bun link` to link it locally for now. To do this, you must clone the Sharkord repository, build it and link the plugin SDK package.

## 5) Create Your Entry File

```ts
import type { PluginContext } from "@sharkord/plugin-sdk";

const onLoad = (ctx: PluginContext) => {
  ctx.log("My Plugin loaded");

  ctx.events.on("user:joined", ({ userId, username }) => {
    ctx.log(`User joined: ${username} (ID: ${userId})`);
  });
};

const onUnload = (ctx: PluginContext) => {
  ctx.log("My Plugin unloaded");
};

export { onLoad, onUnload };
```

## 6) Build the Plugin

Compile to JavaScript before loading into Sharkord:

```bash
bun run build
```

Make sure the `package.json` file is copied to the output directory so Sharkord can read the plugin metadata.
